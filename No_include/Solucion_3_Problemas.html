Now we define our new function which computes the Fibonacci numbers using dynamic programming ( it means that we use the previous results, being the main reason why it is that fast)

In [5]:
def fib_fast(n):
    if n==0:
        return 0
    elif n==1:
        return 1
    elif n>0:
        f_n_1=fib(1)
        f_n_2=fib(0)
        f_n=0
        for i in range(n-1):
            f_n = (f_n_1+f_n_2)
            f_n_2=f_n_1
            f_n_1=f_n
        return f_n

Euler's method for a first-order ODE¶

To do this you only needed to know the way to fill the list and the recursion was given by $ y_{k+1}=y_k+f(x_k,y_k)∗h$, so we will fill the list using the $f$ that was given

In [15]:
import numpy as np
def F_d(x,y):
    return 2.0-np.exp(-4*x)-2*y #the function to fill the list
def solution(x):
    return 1+0.5*np.exp(-4.0*x)-0.5*np.exp(-2.0*x) # the exact solution to compare with

def euler(x_0,y_0,n,T=1): # Euler method which turs out to be only a way to fill the list
    y_list=[]
    x_list=[]
    y=y_0
    for i in range(n+1):
        y_list.append(y)
        x=x_0+i*(T/n)
        x_list.append(x)
        y+=(T/n)*F_d(x,y)
    return np.array(y_list),np.array(x_list)

Here the difficulty was not in the part of filling the list with the solutions of the Euler method, here the real challenge was to truncate the result to 6 decimals, to do this it is necessary to pass the number to string and then take the first 6 decimals, which is enough to take for this case the first 8 numbers, and then pass again the result to a float and return it

In [16]:
def ex_euler(n):
    print(n)
    y_k,x_k=euler(0,1,n)
    z_k=solution(x_k)
    A_k_error=np.abs(y_k-z_k)/z_k
    error=np.sum(A_k_error)/(n+1)
    print("complete error",error)
    result=str(error)[:8]
    print("truncated error",result)
    return float(result)

Strings¶

this one is kind of tricky but it is enough if you understand the rules of the game. since it always have to open and close to return True, we can always check if this is the case and then delete them and check the next. If the string at the end is empty is is because it was well defined.

In [19]:
def validBraces(string):
    while "()" in string or "[]" in string or "{}" in string:
        string=string.replace("()","")
        string=string.replace("[]","")
        string=string.replace("{}","")
    return string==""

def List_of_lists(list_to_work):
    sorted_list=sorted(list_to_work)
    result=[]
    for i in sorted_list:
        result.append([i]*int(i))
    return result

Sorting things in other way:¶

this can be solve easily by using the function of numpy np.binary_repr and using the function count of strings. The solution is given below

In [22]:
import numpy as np
def sort_by_bit(arr): 
    total={}
    a=sorted(arr)
    for i in range(len(a)):
        total[i]=str(np.binary_repr(a[i],32)).count("1")
    result=[]
    for i in sorted(total,key=total.get):
        result.append(a[i])
    return result

or in a shorter and a clever way but making use of a lambda function of python

In [23]:
def sort_by_bit(a):
    return sorted(a, key=lambda x: (bin(x).count("1"), x))

def From_protein_to_rna(Protein):
    result = Protein_Rna_pandas["Stop"] # We take into account the one that says that it stop
    for i in Protein:
        result*=Protein_Rna_pandas[i]
    return result
